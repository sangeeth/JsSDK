JsSDK

AJAX has become the next big thing after Java in the Internet world. AJAX helps to add richness to the web pages and make dynamic web pages appear static, for example: Google's GMail and Google Maps. AJAX is not driven by one single technology. Though the primary technology is JavaScript, technologies like XML, C#, C++, Java, PHP, Perl etc. are being used.

JavaScript is a scripting language widely used for Web Development. Almost all standard web browsers support JavaScript. It helps to add dynamism to the web pages. For developing rich AJAX applications, extensive use of JavaScript is inevitable. JavaScript being a scripting language, the AJAX applications become cumbersome and unmanageable. Cross-Browser JavaScript compatibility is one of the major challenges in Web Development. The web browser when loading a HTML page, downloads the related JavaScript source code as well. It stores the scripts in a temporary directory of the client's machine. Hence it is always risk that valuable code being copied or stolen from the temporary directory.

The most challenging design aspects of an AJAX application are

  • Capability to fall back to default browsing mode (non-JavaScript mode)
  • Retention of User Interface in AJAX mode and non-JavaScript mode
  • Minimal size of JavaScript
  • Easy to develop and maintain.
  • Cross browser compatability

JsSDK comes to rescue by

  • Helping to extend the Model-View-Controller architecture of Web Applications to the client-side JavaScript modules
  • Providing tools to translate Java Server Pages into JavaScript code
  • Providing tools to generate compressed, optimized and obfuscated JavaScript code.
  • Enabling developers to develop commercial quality JavaScript libraries.
  • Providing APIs and tools to dynamically include JSP pages into client-side JavaScript modules.
  • Offering a modified version of DWR technology to access remote Java classes from JavaScript code.
  • Providing Web Browser independent Event Handling APIs
  • Encouraging Object Oriented programming in JavaScript
  • Helping the Java Developer community to build AJAX applications with less effort and cost.

Getting Started

  • Install Java 2 Platform 5.0 SE or higher (Click here)
  • Install any of the following Java IDEs.
    • Sun Java Studio
    • NetBeans
    • Eclipse
    • Oracle JDeveloper
  • Install Apache Ant 1.6.5 or higher (Click here).
  • Install Java 2 Platform 5.0 EE or higher (Click Here)
    Alternatively try using Apache Tomcat 5.5.17 or higher (Click here).

Build Instructions

  • Create a work directory in your machine. For Example: d:\works\JsSDK
  • Unzip JsSDK Source to d:\works\JsSDK.
  • Open Command Prompt.
  • Traverse to d:\works\JsSDK directory.
  • Run: ant dist

The binaries and samples applications are bundled into dist folder. A sample application named Blogger can be located within dist/apps folder. Try it out with any of the J2EE 1.4 compliant servers.
